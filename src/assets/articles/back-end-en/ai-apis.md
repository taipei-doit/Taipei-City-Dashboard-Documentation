## AI Chat System

This project integrates Large Language Models (LLM) from TWS (TWCC) and provides AI chat services through the `langchaingo` framework. The system supports standard conversation, streaming output, and Tool Calling.

### Technical Architecture
- **Framework**: [langchaingo](https://github.com/tmc/langchaingo)
- **Provider**: TWS (TWCC)
- **Concurrency Control**: A semaphore is implemented to limit concurrent AI requests for server stability.
- **Logging**: All AI chat requests and responses (including token usage, latency, etc.) are automatically logged in the `ai_chatlog` database.

---

## Chat Log (ai_chatlog)

For auditing and performance tracking, the backend automatically stores the details of every conversation in the `ai_chatlog` table. 

| Field | Type | Description |
| --- | --- | --- |
| `id` | `int64` | Primary key, auto-incremented. |
| `session_id` | `string` | The session ID of the conversation. |
| `user_id` | `string` | The ID of the user who initiated the request. |
| `provider` | `string` | The AI provider name (default is `twcc`). |
| `model` | `string` | The name of the model used. |
| `question` | `text` | The content of the last message sent by the user. |
| `answer` | `text` | The final reply content generated by the model. |
| `tool_used` | `bool` | Whether any tools were called during the conversation. |
| `tools` | `jsonb` | Details of the tools called (JSON format). |
| `input_tokens` | `int` | Tokens consumed by the input. |
| `output_tokens`| `int` | Tokens consumed by the output. |
| `total_tokens` | `int` | Total tokens consumed. |
| `latency_ms` | `int` | Request latency in milliseconds. |
| `status` | `string` | Request status (e.g., `success`, `error`). |
| `error_code` | `string` | Error code if an error occurred. |
| `error_message`| `text` | Detailed error message if an error occurred. |
| `ip_address` | `string` | Client IP address. |
| `created_at` | `time` | Time when the record was created. |

## Environment Variables

Configure the following variables in your `.env` file to enable AI features:

| Variable | Default | Description |
| --- | --- | --- |
| `TWCC_API_URL` | `https://api-ams.twcc.ai/api` | Base URL for the TWCC API. |
| `TWCC_API_KEY` | - | Your TWCC API Key. |
| `TWCC_MODEL` | `llama3.3-ffm-70b-32k-chat` | The model name to use. |
| `TWCC_TIMEOUT` | `60` | API call timeout in seconds. |
| `TWCC_MAX_RETRY` | `2` | Number of automatic retries on failure (Non-streaming only). |
| `TWCC_MAX_CONCURRENT` | `100` | Maximum allowed concurrent AI requests. |

## APIs

### AI Chat (TWCC)

`POST` `/ai/chat/twai`

This API allows users to converse with the TWCC model, supporting multi-turn dialogue, parameter adjustment, and streaming responses.

| Item | Description |
| --- | --- |
| Permissions | `User` (Login required) |
| Auth | `Authorization: Bearer <JWT_TOKEN>` |

#### Request Body (AIChatInput)

The parameters are designed based on the [TWCC Official API Specification](https://docs.twcloud.ai/docs/user-guides/twcc/afs/api-and-parameters/api-parameter-information).

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `session_id` | `string` | No | Session ID. Automatically generated if not provided. |
| `stream` | `boolean` | No | Enable Streaming mode (SSE). Default is `false`. |
| `messages` | `array` | Yes | List of message history. Each item must have `role` and `content`. |
| `temperature` | `float` | No | Controls randomness (0.0 - 1.0). |
| `max_new_tokens` | `int` | No | Maximum tokens to generate. |
| `top_p` | `float` | No | Nucleus Sampling threshold (0.0 - 1.0). |
| `top_k` | `int` | No | Samples from top K most probable tokens (1 - 100). |
| `frequence_penalty`| `float` | No | Reduces token repetition. |
| `stop_sequences` | `array` | No | List of strings that stop generation (max 4). |
| `seed` | `int` | No | Random seed for stable outputs. |
| `tools` | `array` | No | List of tools for tool calling. |
| `tool_choice` | `any` | No | Specifies the tool choice mode. |

**Example (Complete Parameters):**

```json
{
  "session_id": "test_session_12345",
  "stream": false,
  "messages": [
    { "role": "system", "content": "You are a guide expert for Taipei City." },
    { "role": "user", "content": "Recommend 3 spots in Taipei and tell me the current weather." }
  ],
  "temperature": 0.7,
  "top_p": 0.9,
  "top_k": 50,
  "max_new_tokens": 500,
  "frequence_penalty": 1.1,
  "stop_sequences": ["###", "END"],
  "seed": 42,
  "tools": [
    {
      "type": "function",
      "function": {
        "name": "get_current_weather",
        "description": "Get the current weather in a given location",
        "parameters": {
          "type": "object",
          "properties": {
            "location": { "type": "string", "description": "The city name, e.g. Taipei" }
          },
          "required": ["location"]
        }
      }
    }
  ],
  "tool_choice": "auto"
}
```

#### Response (Non-streaming)

**Success (200 OK):**

```json
{
  "status": "success",
  "data": {
    "session_id": "test_session",
    "content": "Certainly! Here are 3 recommended spots...",
    "usage": {
      "input_tokens": 42,
      "output_tokens": 256,
      "total_tokens": 298
    },
    "tool_used": false,
    "latency_ms": 1850,
    "model": "llama3.3-ffm-70b-32k-chat",
    "provider": "twcc"
  }
}
```

### Advanced Features

#### Streaming Output
When `stream` is set to `true`, the backend returns `text/event-stream`. Content is pushed to the client in real-time as the model generates it.

#### Tool Calling
The API supports intelligent tool calling. If `tools` are defined in the request, the model can decide to call specific tools. The backend executes them and feeds results back to the model (max 5 loops).

#### Concurrency Control (Semaphore)
To prevent server overload, the system uses a Go semaphore for concurrency control. The limit is set via `TWCC_MAX_CONCURRENT`. If reached, the API returns `500 Server too busy`.
